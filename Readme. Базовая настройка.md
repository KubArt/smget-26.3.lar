Справка по базовому ядру проекта (Boilerplate)

Данная база представляет собой готовую экосистему для многопользовательского SaaS-решения с разделением системных и персональных данных.
🛠 Технологический стек

    Backend: Laravel 12.x (PHP 8.2+).

    Frontend: OneUI 5.7 (Bootstrap 5, jQuery 3.7.1).

    База данных: MySQL/PostgreSQL (с поддержкой JSON и полиморфных связей).

    Валидация/Auth: Laravel Breeze (интегрирован и стилизован под OneUI).

🗄 Архитектура базы данных и моделей
1. Расширенная таблица users

   Телефон как ID: В таблицу добавлено поле phone для связи и авторизации.

   Авто-очистка (Mutator): При сохранении любые символы (+, -, ()) удаляются, оставляя только цифры.

   Авто-формат (Accessor): При выводе в интерфейс номер автоматически превращается в формат +7 (XXX) XXX-XX-XX.

2. Таблица user_profiles (One-to-One)

   Отделение данных: Все персональные данные (Имя, Фамилия, Отчество) вынесены в отдельную таблицу для легкости основной таблицы users.

   JSON-хранилище: Поле additional_info с кастом array позволяет хранить любые настройки или мета-данные без изменения структуры БД.

3. Система уведомлений (Polymorphic)

   Универсальность: Используется стандартная таблица notifications, позволяющая отправлять уведомления как конкретному пользователю, так и сущности (например, сайту или проекту).

   Мета-данные в JSON: В уведомлении передаются заголовок, текст, тип (success/danger) и иконка.

⚙️ Ключевые методы и логика
Контроллер профиля (ProfileController)

    Метод update: Использует updateOrCreate, что гарантирует создание записи в user_profiles, если её еще нет.

    Валидация: Реализована проверка уникальности телефона и корректности ввода с выводом ошибок под каждым полем через @error.

Безопасность

    Смена пароля: Реализована проверка current_password (защита от несанкционированного изменения) и строгая валидация нового пароля.

Обработка системных сообщений

    Flash-уведомления: Создан глобальный компонент messages.blade.php, который превращает session('success') во всплывающие уведомления (Toast) через хелпер One.helpers('jq-notify').

🚀 Нюансы для будущих проектов

    Наследование: Все новые контроллеры кабинета должны наследоваться от BaseCabinetController для сохранения общей логики (например, шаринга данных пользователя в шаблоны).

    Плагины: Для работы всплывающих уведомлений обязательно подключение плагина bootstrap-notify.min.js.

    Полиморфизм: Если в новом проекте появляется сущность «Склад» или «Магазин», просто добавь ей трейт Notifiable, и она сможет принимать системные сообщения наравне с пользователем.

📦 Что в архиве?

    Готовая аутентификация (Breeze + OneUI).

    Двухуровневая модель пользователя (User + Profile).

    Интерфейс уведомлений (Header dropdown + Список всех сообщений).

    Страница настроек с валидацией и масками.

    Инструкция по развертыванию:

        Сделать git clone или распаковать архив.

        composer install & npm install.

        Настроить .env и выполнить php artisan migrate --seed.

        Все готово для написания бизнес-логики.
