Создание новых виджетов в нашей текущей архитектуре максимально упрощено благодаря паттерну **Компоновщик**. Чтобы добавить новый блок на дашборд, вам нужно выполнить всего три шага.

---

## 🛠 Инструкция: создание виджета за 3 шага

### Шаг 1: Создание PHP-класса компонента
Создайте новый файл в директории `App\Http\Controllers\Cabinet\Dashboard\`. Класс должен реализовывать интерфейс `DashboardComponentInterface`.

**Пример: `RevenueComponent.php`**
```php
namespace App\Http\Controllers\Cabinet\Dashboard;

use App\Models\User;
use App\Models\Site;
use App\Models\Crm\Lead; // Для примера

class RevenueComponent implements DashboardComponentInterface
{
    // Указываем имя blade-файла (без .blade.php)
    public function getTemplate(): string 
    { 
        return 'revenue_stats'; 
    }

    // Собираем данные для отображения
    public function getData(User $user, ?Site $site = null): array
    {
        $siteIds = $site ? [$site->id] : $user->sites()->pluck('sites.id')->toArray();
        
        // Здесь ваша бизнес-логика
        $revenue = 150000; // Пример суммы

        return [
            'revenue' => $revenue,
            'currency' => '₽'
        ];
    }
}
```

---

### Шаг 2: Создание Blade-шаблона
Создайте файл в папке `resources/views/cabinet/dashboard/widgets/`. Имя файла должно совпадать с тем, что возвращает метод `getTemplate()`.

**Пример: `revenue_stats.blade.php`**
```html
<div class="block block-rounded shadow-sm mt-4">
    <div class="block-content block-content-full d-flex align-items-center justify-content-between bg-gd-primary">
        <div>
            <p class="fs-sm fw-semibold text-white-75 text-uppercase mb-1">Выручка за период</p>
            <p class="fs-2 fw-bold text-white mb-0">
                {{ number_format($revenue, 0, '.', ' ') }} {{ $currency }}
            </p>
        </div>
        <div class="item item-rounded bg-black-10">
            <i class="fa fa-chart-line fs-3 text-white"></i>
        </div>
    </div>
</div>
```

---

### Шаг 3: Регистрация виджета в контроллере
Откройте `DashboardCabinetController.php` и добавьте ваш новый компонент в цепочку вызовов `DashboardManager`.

```php
public function index(Request $request, DashboardManager $manager)
{
    $user = auth()->user();
    $currentSite = $request->site_id ? Site::find($request->site_id) : null;

    $content = $manager
        ->addComponent(new LeadsStatsComponent())
        ->addComponent(new UrgentTasksComponent())
        ->addComponent(new RevenueComponent()) // <-- Просто добавляем новую строку
        ->render($user, $currentSite);

    return view('cabinet.dashboard', [
        'sites' => $user->sites,
        'content' => $content
    ]);
}
```

---

## 💡 Полезные советы

* **Фильтрация:** Всегда используйте переменную `$site` в методе `getData`. Если она не `null`, значит пользователь выбрал конкретный проект, и данные нужно отфильтровать только по нему.
* **Изоляция:** Не пишите стили внутри виджетов. Используйте стандартные классы шаблона **OneUI**, чтобы дашборд выглядел единым целым.
* **Производительность:** Если данных слишком много (например, сложные графики за год), используйте кэширование внутри метода `getData`.
* **Порядок:** Виджеты на странице появятся ровно в том порядке, в котором вы вызываете `addComponent()` в контроллере.

Теперь вы можете расширять аналитику клиники, добавляя любые блоки: от графиков популярности услуг до статистики работы отдельных администраторов.
